# Структура дневника (vault) — ШАБЛОН

Пример организации папок. Адаптируй под себя. Ключевой принцип — разделение
**первоисточника** (то, что ты пишешь/наговариваешь) и **производного** (то, что
генерирует AI).

```
Дневник/
├── 0. Daily Notes/        # ДНЕВНИК — первоисточник, YYYY-MM-DD.md
│                          #   поток сознания, дословно, с пометкой времени
├── 0. Inbox/              # входящее, необработанное
├── 2. Useful/
│   └── People/            # файлы про людей в жизни (первоисточник)
├── 3. Completed/
│   └── Sessions/          # ТРАНСКРИПТЫ СЕССИЙ с терапевтом — первоисточник,
│                          #   не редактировать, внешний источник
├── 4. AI/                 # ПРОИЗВОДНОЕ — можно перегенерировать из первоисточника
│   ├── Parts/             #   карта субличностей (по IFS-логике)
│   ├── Insights Log.md    #   индекс инсайтов (компактный, ≤8000 симв.)
│   ├── Insights/          #   один файл на инсайт, полный голос + ссылки на источник
│   ├── Intentions.md      #   курируемые интенции (векторы внимания)
│   ├── Session Notes/     #   производные заметки по сессиям (со ссылками на транскрипт)
│   └── Summaries by months/  # месячные саммари (сжатая версия старых месяцев)
├── World & self model/    # карты и модели себя (первоисточник)
└── agents.md              # роль и правила AI (см. templates/agents.md)
```

## Первоисточник vs производное

| | Первоисточник | Производное |
|---|---|---|
| Примеры | Daily Notes, транскрипты, People | Parts, Insights, Summaries, Session Notes |
| Кто пишет | пользователь (или транскрипция) | AI |
| Редактирует AI? | **нет** | да, перегенерирует |
| Ссылки | — | ссылается на первоисточник |

## Скользящее окно для загрузки

- **Последние 1–2 месяца** — грузятся **дословно** (полный текст дневников/сессий).
- **Всё старше** — как **месячные саммари** (сжато, с цитатами и ссылками).

Так весь путь всегда в контексте, но влезает в окно модели. См.
[`../scripts/diary-context-bundle.py`](../scripts/diary-context-bundle.py).
