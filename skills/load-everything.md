# Навык: «перезагрузи весь контекст»

Ключевой навык системы. Когда пользователь просит загрузить весь контекст дневника —
это **точная инструкция**, не приглашение к интерпретации. Цель — чтобы весь личный
корпус реально оказался в окне модели за один заход.

## Почему bundle, а не чтение по одному

Наивно читать 250+ файлов по одному — дорого, медленно, и на большом контексте
инструменты начинают сбоить (пустые ответы от насыщения). Правильный способ —
**bundle**: скрипт склеивает весь корпус в несколько больших кусков по ~250k символов,
которые читаются за ~3 команды `cat` вместо 250 обращений.

## Порядок

1. **Пересобрать bundle** (подхватывает новые файлы и саммари):
   ```
   python3 scripts/diary-context-bundle.py
   ```
   Порядок сборки: фреймы → части → инсайты → люди → месячные саммари (старые месяцы)
   → полный текст сессий+дневников за последний полный и текущий месяц (по датам).
   Скрипт сам считает окно от сегодняшней даты.

2. **Прочитать каждый кусок через `cat`, по порядку** (НЕ через read-tool — у него
   лимит; `cat` отдаёт кусок целиком):
   ```
   cat <bundle>/chunk-01.md
   cat <bundle>/chunk-02.md
   ...
   ```

3. **Манифест** — полный список файлов и порядок в `MANIFEST.txt`.

## Правила выполнения

1. Не сокращать. Не пропускать. Не делать дайджест. Если у пользователя много
   контекста — у AI тоже.
2. Не полагаться на авто-загрузку воркспейса (она режет при превышении лимита).
   Обязательные файлы читать явно.
3. Не предлагать альтернативы («сжать ранние месяцы», «только ключевое»), если не
   просили. Это микро-срезание углов под видом эффективности.
4. Если упёрся в реальный лимит — сказать прямо: «загружено по дату X, дальше
   обрезается», решение — за пользователем.

## Зачем
Это рабочий инструмент для глубоких сессий и плотной работы. Воспроизводимый результат:
~9 обращений вместо 250, весь путь в контексте, без пустых ответов.
