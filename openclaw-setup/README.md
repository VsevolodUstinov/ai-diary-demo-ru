# OpenClaw-слой: из чего складывается агент

Дневниковый vault — это **контент**. Чтобы с ним работал живой агент, нужен слой
**личности + контекста + ритмов**. Здесь — из каких файлов это складывается и
обезличенные скелеты, чтобы читатель с агентом собрал свою версию.

> Реальные файлы личности/контекста автора (имя агента, память, инфраструктура) —
> приватные и не публикуются. Здесь только **concept + скелеты**. Наполняешь под себя.

## Слои агента

| Файл | Что задаёт | Пример в репо |
|---|---|---|
| `IDENTITY.md` | Имя, образ, роль, тон конкретного агента | [IDENTITY.md](./IDENTITY.md) — **реальный** |
| `SOUL.md` | Характер, ценности, границы — «каким быть помощником» | [SOUL.md](./SOUL.md) — реальный (общий) |
| `USER.md` | Кто владелец, что важно, контекст | *(приватно — концепт ниже)* |
| `MEMORY.md` | Долговременная память (факты, решения, предпочтения) | *(приватно — концепт ниже)* |
| `agents.md` | Правила работы именно с дневниковым vault | [`../vault/agents.md`](../vault/agents.md) |
| cron | Ритмы: утренний апдейт, недельный check-in, авто-обработка | [cron-examples.md](./cron-examples.md) |
| skills | Навыки: загрузка контекста, обработка сессий | [`../skills/`](../skills/) |

## Как это работает вместе

```
SOUL + IDENTITY  →  кто такой агент (характер + имя/роль/тон)
USER + MEMORY    →  кого он знает (владелец + накопленный контекст)
agents.md        →  как работать с дневником (роль-зеркало, правила)
vault/           →  сам контент (дневники, части, инсайты, сессии)
cron + skills    →  ритмы и инструменты (утро, check-in, «загрузи всё»)
```

Агент при старте читает слои личности/контекста + `agents.md`, при глубокой работе —
подтягивает весь vault через bundle («перезагрузи всё»).

## Про USER.md и MEMORY.md (приватные — только концепт)

**USER.md** — кто владелец: имя, контекст, что важно, как помогать. Слишком личный,
чтобы публиковать — но принцип простой: то, что агент знает про тебя с самого старта,
чтобы не объяснять с нуля. Не досье — рабочий контекст.

**MEMORY.md** — долговременная память: курируемая выжимка (значимые решения, предпочтения,
технические факты сетапа, уроки). В отличие от дневника (поток сознания) — это дистиллят,
«как долговременная память человека». Принцип — то же разделение источник/производное,
что и в vault. Содержание слишком специфично для каждого, поэтому не публикуется.

## Что нужно читателю, чтобы собрать свою версию

1. **Агент-оркестратор** с доступом к папке (чтение/запись), cron, exec, skills.
2. **Слои личности/контекста** — IDENTITY (имя/роль/тон) + SOUL (характер) + USER/MEMORY
   (про владельца — заполнить самому).
3. **`agents.md`** в корне vault — роль-зеркало + правила (взять из `vault/agents.md`).
4. **Vault** — структура папок (см. `vault/`).
5. **Ритмы** — cron на утренний апдейт, недельный check-in, авто-обработку транскриптов.
6. **Навыки** — «загрузи весь контекст», обработка сессий (см. `skills/`, `prompts/`).
