# Cron-ритмы (обезличенные примеры)

Ритмы, которые работают сами и держат процесс между разговорами. Формат зависит от
твоего агента; здесь — назначение и логика, а не точный синтаксис.

## 1. Утренний апдейт (каждое утро)

Соединяет объективные данные тела с субъективным дневником.

```
Триггер: cron, каждое утро (напр. 08:00 локального времени)

Шаги:
1. Скрипт тянет цифры из Whoop API   → recovery, сон, sleep debt, HRV
2. Скрипт берёт локацию + трекинг-поля (напр. sleep debt)
3. AI читает свежие инсайты / интенции → что важно сейчас напомнить
4. AI пишет живой апдейт ЧЕРЕЗ мотивацию владельца
   (не «спи», а «это в сторону хорошего дня»)
```

**Важно:** цифры тянет **скрипт** (детерминированно, без галлюцинаций), текст пишет
**AI** (тёпло, через мотивацию владельца, не через тревогу). Объективный график тела
оказывается рядом с субъективным состоянием — часто объясняет самочувствие раньше, чем
человек сам заметит.

## 2. Воскресный check-in (раз в неделю)

Короткий ритуал безопасности. Даёт тревожной части предсказуемость.

```
Триггер: cron, воскресенье
Действие: AI задаёт 3 договорённых вопроса (риск / игнор сигналов тела / самочувствие),
          фиксирует ответ в дневник.
```

Если check-in пропущен несколько недель подряд — это само по себе сигнал. См.
[`../prompts/weekly-checkin.md`](../prompts/weekly-checkin.md).

## 3. Авто-обработка транскрипта сессии (по событию/расписанию)

Когда появляется новый транскрипт сессии с терапевтом — агент обрабатывает его в
производную заметку.

```
Триггер: новый файл в папке транскриптов (или ручной запрос)
Действие: AI по промпту process-transcript генерирует Session Note
          со ссылками, кандидатами в инсайты, обновлениями частей.
          Инсайты в курируемую карту — только с подтверждением владельца.
```

См. [`../prompts/process-transcript.md`](../prompts/process-transcript.md).

## Принцип

Cron — для точного времени и standalone-задач (утро, воскресенье, обработка файла).
Разговоры — для живого контакта. Скрипт — для детерминированных данных. AI — для тёплого
текста и связей. Разделение ответственности держит систему честной.
