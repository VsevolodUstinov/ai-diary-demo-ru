# AI-дневник как терапевтический со-пилот

Полгода живого опыта: как разговор с AI, у которой есть **весь** мой контекст —
моим прямым текстом и транскриптами сессий с терапевтом — меняет работу над собой
между встречами.

Кейс для выступления на конфе про AI в медицине. Формат: self-therapy + human
therapist в петле. Стек: **OpenClaw + Obsidian**.

👉 **Презентация (открывается как страница):** [index.html](./index.html) —
включите GitHub Pages, чтобы получить живую ссылку
(`Settings → Pages → Branch: main → /root`).

## Философия в 5 принципах

1. **Главное — говорить с AI, у которой РЕАЛЬНО есть контекст.** Не пересказ в двух
   абзацах, а мой прямой текст (дневники) + полные транскрипты сессий. Первоисточник,
   не пересказ.
2. **Не страшно, что это 300k токенов за раз.** Окно ~1M позволяет грузить весь личный
   корпус целиком — это фича, а не проблема. AI видит все связи разом.
3. **Роль — зеркало, а не поддакиватель.** Явный список «видов фейка», которых нельзя
   делать (слепая поддержка, маршрутизация к терапевту, инфляция важности, покровительство).
4. **Исходные данные ≠ сгенерированное.** Первоисточник (дневники, транскрипты)
   неприкосновенен. Производное (карты, сводки) можно перегенерировать, и оно ссылается
   на источник.
5. **Курируемые карты меняются только с подтверждением.** Инсайты/интенции — моя карта,
   не интерпретации AI. AI предлагает, я решаю.

## ⚠️ Приватность

Настоящий дневник — **приватный** и таким остаётся. Здесь — только **механика**:
структура, роль, промпты, архитектура. Примеры диалогов реальные, но обезличенные;
темы травм, зависимостей и любая информация о других людях — не показываются.

Если строишь такое для себя: держи первоисточник локально/приватно, а публично делись
только обвязкой. Система должна выдерживать настоящую интимность — иначе она бесполезна.
